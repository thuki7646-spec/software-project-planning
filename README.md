1. Жобаны жоспарлау

Бұл жобаны орындау барысында бағдарламалық өнімді әзірлеудің заманауи әдістемесі — Scrum қолданылды. Scrum әдістемесі командалық жұмысты тиімді ұйымдастыруға және жобаны кезең-кезеңімен дамытуға мүмкіндік береді. Жоба 2 апталық спринттерге бөлініп, әр спринт соңында орындалған жұмыстар талданып отырды. Күн сайын 15 минуттық күнделікті жиналыс (Daily meeting) өткізу жоспарланды. Әр спринт аяқталғаннан кейін ретроспектива жүргізіледі.

Scrum артефактілері ретінде өнім бэклогы қолданылды. Бэклогта жобаның барлық функционалдық талаптары мен міндеттері көрсетілді. Жобаны басқару үшін Jira немесе GitHub Projects платформаларын қолдану жоспарланды.

2. Командадағы рөлдер

Жобаны тиімді орындау үшін келесі рөлдер анықталды:

Project Manager / Scrum Master — жобаның орындалуын бақылау, мерзімдерді қадағалау және команданың жұмысын үйлестіру.

System Analyst — жүйеге қойылатын талаптарды талдау және API спецификацияларын сипаттау.

Backend Developer — серверлік логиканы және мәліметтер базасымен жұмыс істеуді жүзеге асыру.

Frontend Developer — қолданушы интерфейсін жасау.

DevOps Engineer — серверлік инфрақұрылымды, CI/CD процестерін және автоматтандыруды баптау.

3. Жобаның іске асыру кезеңдері (Roadmap)

Жобаны іске асыру келесі кезеңдер бойынша жоспарланды:

1-ай: жүйенің архитектурасын жобалау, технологиялық стек таңдау және мәліметтер базасының құрылымын анықтау.

2-ай: өнімнің ең төменгі жұмыс істейтін нұсқасын (MVP) әзірлеу.

3-ай: тестілеу жүргізу, анықталған қателерді түзету және өнімнің 1.0 нұсқасын релиз жасау.

4. Бағдарламалық инфрақұрылымды ұйымдастыру

Жобада 2025 жылы кеңінен қолданылатын Cloud-Native тәсілі пайдаланылды. Нұсқаларды басқару жүйесі ретінде GitHub немесе GitLab қолданылды. Қосымшаның орындалу ортасы Docker контейнерлері арқылы жүзеге асырылады. Қосымшаны масштабтау үшін Kubernetes оркестрация жүйесін қолдану жоспарланды.

5. CI/CD процесін ұйымдастыру

Жобада үздіксіз интеграция және үздіксіз жеткізу (CI/CD) процесі келесі алгоритм бойынша жұмыс істейді:

Әзірлеуші кодты репозиторийге жүктейді (commit).

Автоматты түрде тестілеу және код сапасын тексеру жүргізіледі.

Қауіпсіздік сканері арқылы осал тұстар тексеріледі.

Docker бейнесі құрастырылып, репозиторийге жіберіледі.

Қосымша тестілеу ортасына (Staging) автоматты түрде орналастырылады.

6. Орталар және мониторинг

Жоба үш ортада жұмыс істейді:

Dev — әзірлеушілерге арналған орта.

Stage — тестілеу және тексеру ортасы.

Prod — қолданушыларға арналған негізгі орта.

Жүйенің жұмысын бақылау үшін Grafana мониторинг жүйесі және қателерді бақылау үшін Sentry сервисін қолдану жоспарланды.

7. Қауіпсіздік

Қауіпсіздікті қамтамасыз ету мақсатында құпия деректерді сақтау үшін арнайы шешімдер қолданылды. Парольдер мен API кілттерді сақтау үшін HashiCorp Vault немесе GitHub Secrets пайдалану жоспарланды.
